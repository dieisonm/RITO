# Backlog de Implementação de Aquisição

Data: `2026-05-08`
Status: `proposto`

## Prioridade 1. Medição

Objetivo: saber quais canais geram conversa real.

Tarefas:

- confirmar ou criar propriedade no Google Search Console
- confirmar ou criar GA4
- decidir se usaremos Google Tag Manager
- adicionar tags ao site quando os IDs estiverem definidos
- criar eventos:
  - `generate_lead_form`
  - `click_whatsapp`
  - `click_email`
  - `view_contact_page`
- padronizar UTMs:
  - `utm_source`
  - `utm_medium`
  - `utm_campaign`
  - `utm_content`

Critério de pronto:

- formulário e cliques de WhatsApp aparecem como eventos rastreáveis
- campanhas conseguem apontar para uma origem identificável

## Prioridade 2. Landing pages

Objetivo: parar de mandar todo tráfego para uma mensagem genérica.

Páginas sugeridas:

- `/pages/software-sob-medida-pequenas-empresas.html`
- `/pages/automacao-atendimento-rotina.html`
- `/pages/dashboards-controles-internos.html`
- `/pages/site-para-negocios-locais.html`

Cada página deve ter:

- promessa específica
- exemplos de rotinas que a RITO pode organizar
- CTA para WhatsApp e formulário
- FAQ curta
- prova de confiança institucional

## Prioridade 3. Google Ads piloto

Objetivo: testar demanda ativa.

Estrutura:

- uma campanha de pesquisa
- grupos por intenção:
  - software sob medida
  - automação para pequena empresa
  - dashboard e controle
  - site para pequeno negócio
- orçamento pequeno
- leitura mínima de 7 a 14 dias

Critério de pausa:

- cliques sem permanência
- cliques sem ação de contato
- termos de busca muito fora da oferta

## Prioridade 4. Meta / Instagram Ads piloto

Objetivo: aquecer público e gerar familiaridade.

Regras:

- não acessar perfis de prospects
- não automatizar Instagram
- usar somente criativos próprios e Ads Manager
- testar criativos educativos antes de criativos comerciais diretos

Criativos iniciais:

- rotina no improviso
- planilha + WhatsApp + retrabalho
- software sob medida sem custo corporativo pesado
- dashboards e controles internos para pequenas empresas

Critério de pronto:

- 3 a 5 peças aprovadas
- página de destino definida
- evento de conversão funcionando

## Prioridade 5. Outbound com aprendizado

Objetivo: usar prospecção ativa com mais precisão.

Mudanças:

- cada lote precisa declarar hipótese
- cada lote precisa ter variação de oferta ou segmento
- não repetir batch multissetorial genérico sem aprendizado
- priorizar segmentos que reagirem em SEO, Ads ou conteúdo
