# Site Backlog da RITO Sistemas

Este diretorio organiza o backlog de melhorias do site da RITO Sistemas a partir das auditorias e recomendacoes ja produzidas em `docs/website/audits/` e `docs/website/plans/`.

## Objetivo

Transformar as observacoes sobre o site em trabalho executavel, com foco em:

- clareza da proposta
- conversao de contato
- hierarquia da informacao
- prova de confianca
- consistencia de copy

## Escopo deste backlog

Este backlog cobre apenas conteudo e estrutura editorial do site.

Nao faz parte deste diretorio:

- alteracao de HTML, CSS ou JS
- mudanca de layout em codigo
- publicacao automatica no site
- decisao juridica ou fiscal final

## Fontes de verdade usadas

- `docs/website/audits/site-ux-conversion-audit.md`
- `docs/website/audits/contact-flow-audit.md`
- `docs/website/audits/information-architecture-review.md`
- `docs/website/plans/homepage-copy-recommendations.md`
- `docs/website/plans/internal-pages-copy-recommendations.md`
- `docs/website/plans/contact-page-copy-strengthening.md`

## Arquivos deste diretorio

- `site-improvement-backlog.md`: fila de tarefas priorizada com dono, dependencias, risco e criterio de pronto.
- `implementation-sequence.md`: ordem recomendada para executar o backlog sem criar retrabalho.
- `content-assets-needed.md`: lista de materiais e textos de apoio que destravam a execucao.
- `acceptance-checklists.md`: checklists de aceite para revisar os entregaveis antes de considerar cada item pronto.

## Como usar

1. Comece por `site-improvement-backlog.md` para escolher o proximo item.
2. Use `implementation-sequence.md` para respeitar dependencias entre copy, arquitetura e prova.
3. Consulte `content-assets-needed.md` para produzir os insumos que faltam.
4. Valide o resultado com `acceptance-checklists.md`.

## Regra principal

Nada neste diretorio altera o site diretamente. Ele serve para organizar o trabalho antes de qualquer implementacao futura.
