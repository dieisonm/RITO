# Checklist de Envio da Proposta

## Objetivo

Garantir que a proposta saia coerente, completa e alinhada ao que foi combinado com o cliente.

## Antes de enviar

### 1. Conteúdo

- [ ] O nome do cliente está correto?
- [ ] O responsável está correto?
- [ ] O problema principal foi descrito com clareza?
- [ ] O objetivo da solução está coerente com o briefing?
- [ ] O escopo inclui apenas o que foi combinado?
- [ ] O que não está incluído está claro?
- [ ] O prazo estimado faz sentido?
- [ ] O investimento está alinhado com o nível da demanda?

### 2. Forma

- [ ] O texto está em português do Brasil?
- [ ] O tom está profissional e simples?
- [ ] Não há linguagem técnica desnecessária?
- [ ] Não há promessa exagerada?
- [ ] A proposta parece operacional, não conceitual?

### 3. Comercial

- [ ] A análise inicial grátis e sem compromisso está clara?
- [ ] O próximo passo está definido?
- [ ] A forma de pagamento está preenchida?
- [ ] A validade da proposta foi informada?
- [ ] O follow-up já foi pensado caso o cliente não responda?

### 4. Anexos e apoio

- [ ] O briefing está anexado ou referenciado?
- [ ] A estimativa foi incluída, se necessário?
- [ ] Há observações de escopo que precisam acompanhar o envio?
- [ ] O cliente recebeu apenas o que precisa ver agora?

## Texto de envio padrão

```text
Oi, [NOME]. Tudo bem?

Segue a proposta com base no que entendemos até aqui.
Se quiser, eu também posso esclarecer qualquer ponto do escopo, prazo ou investimento.
```

## Se o cliente ainda estiver inseguro

Use este ajuste:

```text
A proposta foi montada com base no briefing que recebemos. Se você quiser, podemos ajustar qualquer ponto antes de seguir.
```

## Depois do envio

Registrar:

- data de envio;
- canal usado;
- versão da proposta;
- prazo combinado para retorno;
- próximo follow-up.

## Regra final

Se houver dúvida relevante de escopo, não enviar ainda. Corrigir antes de encaminhar.
