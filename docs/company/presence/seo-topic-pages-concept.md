# Conceito das paginas SEO especificas da RITO

Data: 2026-05-20

## Objetivo

Criar paginas organicas para buscas especificas sem transformar o site em um bloco longo de texto. Cada pagina precisa explicar rapidamente o que a RITO faz, mostrar exemplos visuais e conduzir para contato.

## Principio de experiencia

As paginas devem parecer uma demonstracao, nao um artigo. O visitante precisa bater o olho e entender:

- qual problema aquela solucao resolve;
- que tipos de entregas a RITO pode fazer;
- como isso se parece na pratica;
- qual proximo passo tomar.

## Paginas planejadas

1. Software sob medida para pequenas empresas
2. Automacao para MEI e pequenos negocios
3. Planilhas inteligentes e dashboards sob medida
4. Criacao de site simples para MEI

## Estrutura recomendada para cada pagina

### 1. Hero objetivo

- Titulo com a palavra-chave principal.
- Subtitulo curto explicando o ganho pratico.
- CTA primario para contar a rotina ou pedir analise.
- CTA secundario para ver exemplos na pagina.

### 2. Carrossel visual de possibilidades

Um componente horizontal ou em abas com mockups como:

- dashboard simples de vendas ou pedidos;
- controle de clientes e retornos;
- tela de formulario inteligente;
- planilha inteligente com indicadores;
- pagina/site simples com CTA de contato.

Objetivo: reduzir dependencia de texto e mostrar tangivelmente o que "software sob medida" significa.

### 3. Bloco "quando faz sentido"

Cards curtos com situacoes reais:

- informacoes espalhadas em WhatsApp, papel e planilha;
- dificuldade de acompanhar pedidos, prazos ou retornos;
- retrabalho com copia e cola;
- falta de visao rapida de indicadores;
- site antigo, inexistente ou pouco claro.

### 4. Antes e depois

Comparacao visual simples:

- antes: planilha solta, mensagem perdida, controle manual;
- depois: controle centralizado, painel, fluxo claro, rotina acompanhavel.

### 5. Exemplos de entregas

Cards de entregas possiveis, com linguagem de negocio:

- controle de pedidos;
- controle de clientes;
- painel financeiro simples;
- automacao de orcamentos;
- formulario de entrada de demandas;
- site simples para receber contatos.

### 6. Como a RITO trabalha

Explicacao curta em 3 passos:

- entender a rotina;
- desenhar uma primeira solucao viavel;
- construir, testar e evoluir.

### 7. CTA final

Convite para a pessoa contar qual rotina quer organizar. Usar linguagem simples e direta.

## Direcao visual

- Continuar com a identidade RITO: azul petroleo, areia, dourado e branco.
- Usar mockups em tablets, notebooks ou cards flutuantes.
- Evitar telas falsas muito corporativas; a RITO fala com negocios pequenos, entao os exemplos precisam parecer acessiveis.
- Balancear visual premium com clareza pratica.
- Pouco texto por dobra, mais blocos de exemplo.

## Hipotese de design

Uma pagina com cara de "galeria de solucoes" pode converter melhor do que uma pagina institucional longa, porque o cliente ainda nao sabe exatamente o que pedir. Ao ver exemplos visuais, ele consegue reconhecer uma dor propria e iniciar contato com mais facilidade.

## Cuidados de SEO

- Cada pagina deve ter um H1 unico com a intencao principal.
- Evitar repetir o mesmo texto nas quatro paginas.
- Manter conteudo visivel suficiente para o Google entender a pagina.
- Usar dados estruturados apenas quando refletirem conteudo visivel.
- Interatividade nao pode esconder tudo do HTML; carrossel/abas devem ter conteudo indexavel.
