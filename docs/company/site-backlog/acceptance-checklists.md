# Acceptance Checklists

## Como usar

Use estas checklists para revisar cada entregavel antes de considerar o item pronto.

## Homepage

- [ ] A headline explica a proposta da RITO em linguagem simples.
- [ ] O subtitulo deixa claro para quem a empresa trabalha.
- [ ] O CTA principal aponta para contato ou analise inicial.
- [ ] Existe uma faixa curta de confianca logo no inicio.
- [ ] A pagina mostra problema, solucao e resultado pratico.
- [ ] O texto nao depende de buzzwords ou linguagem de startup.
- [ ] O visitante entende o que fazer depois de ler a pagina.
- [ ] O mobile continua facil de escanear.

## Paginas internas

- [ ] Cada pagina tem uma funcao primaria unica.
- [ ] Solucoes mostra exemplos concretos e nao apenas categorias.
- [ ] Como Funciona explica as etapas do atendimento.
- [ ] Para Quem E qualifica o publico com mais clareza.
- [ ] Sobre explica criterio de trabalho e posicionamento.
- [ ] As paginas nao repetem a homepage inteira.
- [ ] Os CTAs sao coerentes entre as paginas.
- [ ] O tom continua padrao em todo o site.

## Pagina de Contato

- [ ] A pagina deixa claro que a analise inicial e gratuita.
- [ ] O texto de apoio reduz a friccao de quem ainda nao tem escopo fechado.
- [ ] Fica claro o que acontece depois do envio.
- [ ] Existe indicacao objetiva de canal ou expectativa de retorno.
- [ ] O formulario nao pede mais do que o necessario na triagem inicial.
- [ ] O FAQ tira as duvidas mais comuns antes do envio.
- [ ] A confirmacao de envio comunica proximo passo de forma serena.

## Prova e Conteudo

- [ ] Existem exemplos de solucao que parecem plausiveis para pequenas empresas.
- [ ] Existem casos ficticios com contexto, problema, solucao e resultado esperado.
- [ ] Existem antes/depois de processos que mostrem ganho pratico.
- [ ] Os materiais deixam claro que sao ilustrativos quando necessario.
- [ ] A prova pode ser reaproveitada em site, deck e vendas.
- [ ] O material nao promete entrega real que ainda nao existe.

## Consistencia comercial

- [ ] A frase `orcamento gratis e sem compromisso` aparece com consistencia.
- [ ] A marca usa sempre o mesmo nome institucional.
- [ ] O tom e claro, profissional e direto.
- [ ] A comunicacao reforca pequenos negocios e problemas reais.
- [ ] Nao ha promessa de menor preco do mercado.
- [ ] Nao ha linguagem tecnica excessiva.
- [ ] A empresa parece organizada e facil de acionar.

## Implementacao visual e front-end

- [ ] O escopo foi classificado como `granular_ui_change` ou `responsive_ui_build`.
- [ ] A rota, viewport e componente afetado foram identificados antes da edição.
- [ ] A mudança reutiliza componentes, tokens, ícones e padrões existentes.
- [ ] O comportamento, os dados e a navegação continuam iguais quando a mudança é apenas visual.
- [ ] A alteração foi verificada em navegador real.
- [ ] Desktop e mobile foram conferidos quando o layout foi afetado.
- [ ] Não houve redesign amplo disfarçado de ajuste pequeno.
- [ ] As premissas visuais ou ambiguidades foram registradas.

## Aceite final do backlog

- [ ] A jornada do site sai de explicacao para conversao.
- [ ] A homepage orienta a leitura.
- [ ] As paginas internas aprofundam sem repetir.
- [ ] O contato reduz ansiedade e aumenta previsibilidade.
- [ ] A prova ajuda a sustentar a decisao do visitante.
- [ ] O backlog pode ser entregue para revisao humana sem pendencias obvias.
- [ ] Quando houve código, a validação visual e técnica foi registrada.
